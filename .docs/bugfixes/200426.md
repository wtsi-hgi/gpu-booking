# Bugfix Checklist

- [x] No ability to close new booking view, have to use browser back
- [x] Should have a dark mode that is used automatically
- [x] Using the configured admin user (sb10@sanger.ac.uk), there doesn’t seem any way to access any admin interface to process booking requests; has it even been implemented?
- [x] There’s no way to drag-select very long bookings, such as more than 1.5 months. Could it automatically reveal the next month underneath if you drag beyond the last day (which is already the first week or so of the next month, so avoid duplications)
- [x] Don’t show “Cancelled” state bookings anywhere in the normal interface; only in admin screens.
- [ ] ./run-dev.sh should be refactored to a Makefile that has lint, format, test, run actions. Run can call run-dev.sh, which should only do the minimum needed to bring up the backend and frontend (ie. not run format, lint and tests).
- [ ] Event start and end date should default to the booking start and end, not today
- [ ] There are no playright E2E tests. Implement them to lock in all current behaviour (which should both cover all end-user interface click behaviour implied by `.docs/booking/spec.md`, but also effectively result in regression tests for some of the other issues above), and have them run as part of `make test`. Do not install Playwright or its dependencies on this server. Node, pnpm, and Playwright browsers are already installed and configured via PATH and PLAYWRIGHT_BROWSERS_PATH. Do not run playwright install, playwright install --with-deps, apt, sudo, or any bootstrap step. Use the existing environment and only run project-local dependency installation if the frontend itself requires its own package lock resolution. When running ad hoc Playwright tests without a config, use --browser=chromium, not --project=chromium.
- [ ] Okta implementation does not seem complete; frontend doesn’t use or forward OIDC_* env var details to backend which doesn’t seem to use such details.
