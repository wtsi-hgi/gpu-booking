# Bugfix Checklist

- [x] `make run` should have a documented ability to take FRONTEND_PORT and BACKEND_PORT from a `.env` file. These and any other env vars in use should also be renamed to avoid conflicts with other systems, eg. GPU_BOOKING_FRONTEND_PORT etc.
- [x] `make test` is failing. Node, pnpm, and Playwright browsers are already installed and configured via PATH and PLAYWRIGHT_BROWSERS_PATH. Get the tests to use these correctly.
- [x] The “Close” button on the Create Booking form should not be next to the “Create Booking” button. It’s too easy to make a mistake. The close ability should be a standard close icon on the overall “Create Booking” form box, and should ask for verification to close if any of the form details have been altered from its initial appearance state.
- [x] The “Create Booking” form is in a box underneath a “New Booking” header, which is redundant. Get rid of “New Booking” header and “Start a booking request by choosing a date range.” line.
- [x] In dark mode, the highlight of selected days in calendar views is not really visible. Nor is the border on the “Selection details” box.
- [x] Switching from Calendar view to Table view loses the indication of what month of bookings we are viewing, and loses the ability to change month. Table view should just show all current and future bookings and not be limited by the month selected on the Calendar view.
- [x] In Calendar view, instead of “Previous Month” and “Next Month” buttons, there should be left and right arrow buttons surrounding the display of the month, and the month word itself should be clickable to reveal a month selector. Likewise the year should have left and right arrow buttons (but not be clickable).
- [x] In Calendar view, the “Today” button should set the month to the current month and animate a brief highlight of the current day.
- [x] A cancelled booking immediately dissapears from the Table view, but then reappears when switching to Calendar view, and is visible again going back to Table view. It only goes away properly after manually refreshing the page. A manual refresh should not be necessary.
- [x] Admin editing of a booking to change it to confirmed shows a “Capacity warning: confirmed bookings consume GPU capacity. Review availability before saving.”, even when there’s no capacity issue and no other bookings in any other state for those days. The warning stays there after refresh viewing the confirmed booking. Only warn when choosing to confirm would put over capacity.
- [x] Having entered the Admin Dashboard, the “Admin Dashboard” button should change to “Bookings” button to allow going back to the default view.
